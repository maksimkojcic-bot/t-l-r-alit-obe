Use this file to avoid repeating mistakes. When the user corrects Codex, add a new dated entry.

## 2026-07-17 - Wrong Visual Direction

What went wrong:

- Codex interpreted "pousse sur le gras / souille" as dirty, grimy, grotesque horror-style visuals.
- The generated V3 looked too much like a static grossout still image with shake.

Corrected rule:

- The target is clean glossy 3D reality TV: luxury villa, pool, palm trees, neon signs, saturated color, fruit characters with very large cartoon bellies.
- "Pousse" means exaggerated fruit bodies, attitude, clash, comedy, and reality TV energy, not filthy horror visuals.

How to avoid it:

- Read `docs/reference-0605-style-lock.md` before generating visuals.
- Use the user's `0605(4).mov` reference as the style lock.
- Generate one validation frame before spending credits on video APIs.

## 2026-07-17 - Static Motion Is Not Enough

What went wrong:

- Remotion-only animation created the feeling of a photo moving, not a real animated scene.

Corrected rule:

- Production quality requires image-to-video clips from Kling, Veo, or an equivalent model.
- Remotion is for final edit, transitions, timing, sound, captions, and export.

How to avoid it:

- Do not present a still-image Remotion test as final quality.
- Plan each episode as multiple short animated clips, then assemble them in Remotion.

## 2026-07-17 - Subtitle Background

What went wrong:

- A black subtitle/background style was used in a test.

Corrected rule:

- Subtitles should be bold white text with black stroke only.
- Do not use a black rectangle behind subtitles.

How to avoid it:

- If burning subtitles in Remotion, use text shadow/stroke only.
- Remember CapCut Pro can also create final auto-captions.
